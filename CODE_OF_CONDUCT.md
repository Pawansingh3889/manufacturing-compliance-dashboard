# Code of conduct

Be useful, be honest, don't be a jerk. This project is small and solo-
maintained; the code of conduct is set low and clear on purpose.

## Expected behaviour

- **Useful** — reports have reproduction steps. Feature requests
  describe the problem, not just the solution. Reviews suggest
  specific changes, not general dissatisfaction.
- **Honest** — if you don't know, say so. If you benchmarked, show
  numbers; if you didn't, say "I think" instead of "this is faster".
- **Respectful of the audience** — the dashboard's users are shift
  leads, QA technicians, and auditors. Things that are obvious to a
  Python developer may not be obvious to them.
- **Good-faith reading** — most rude-sounding comments on GitHub are
  someone on a phone between meetings. Ask before assuming hostility.

## Not tolerated

- **Personal attacks** — on the maintainer, contributors, or anyone
  CC'd in an issue or PR.
- **Harassment of any kind** — targeted pile-ons, unwelcome sexual
  attention, doxxing. Zero tolerance, no warning.
- **Commercial spam** — this is not a place to promote unrelated
  SaaS or tooling.
- **Bad-faith reports** — filing issues to waste maintainer time is
  harassment dressed as contribution.

## Scope

Applies to issues, PRs, discussions, code review comments, and public
communication about the project on external platforms. Does not apply
to private messages between individuals or unrelated projects.

## Reporting

Abuse is reported through the same private channel as security:

<https://github.com/Pawansingh3889/manufacturing-compliance-dashboard/security/advisories/new>

Mark the subject "Code of conduct" so it doesn't get confused with a
vulnerability report. Acknowledgement within 7 days.

## Enforcement

Handled by the maintainer in three tiers:

1. **Private warning** — note explaining what was unacceptable.
2. **Temporary block** — repeated warnings or one serious incident;
   loss of repo interaction for days to weeks.
3. **Permanent block** — harassment that resumes after a temporary
   block, or any threats / doxxing.

Rulings are judgement calls, made in public where appropriate and in
private when the target prefers privacy.
